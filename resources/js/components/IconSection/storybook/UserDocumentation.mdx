# Icon Section

The **Icon Section** block presents a set of visual icons paired with short descriptions.

It is designed to highlight key features, services, values, or benefits in a clear and visually structured way. The block combines an introductory context area with a grid of icon-based items.

---

## When to Use This Block

Use the Icon Section when:

- You want to highlight multiple key ideas or benefits
- You need a visually engaging grid of icon-based content
- You want to present information in a quick, scannable format
- You are building feature or service overview sections

### Example uses:

- Key program highlights
- Institutional values or pillars
- Service or benefit summaries
- Feature explanations
- Process or step-based introductions

## Block Structure

The Icon Section block contains two primary areas:

1. **Context Section**
2. **Icon Grid**

The context introduces the section, while the icon grid displays individual items with icons and supporting text.

---

## Context Section

The context area appears at the top of the block and introduces the section.

It may include:

- **Subheading (optional)** – A short introductory label
- **Heading** – The main title of the section
- **Paragraph** – Supporting description or explanation
- **Buttons (optional)** – Calls to action related to the section

The context is centered and designed to provide clear framing for the icon items that follow.

---

## Icon Grid

Below the context area is a grid of icon items.

Each item includes:

- **Icon image**
- **Heading**
- **Short paragraph**

Icons appear above their associated text and are centered within the layout.

The grid automatically adjusts based on screen size.

---

## Adding Icon Items

Editors can add multiple icon items using the **Items** field.

Each item requires:

### Image

The icon image displayed above the content.

Tips:

- Use simple, clear icons
- Maintain consistent style across all icons
- Use square images for best results

---

### Heading

A short title describing the item.

Examples:

- Fast Application Process
- Student Support
- Career Opportunities

Keep headings concise for better readability.

---

### Paragraph

A short description explaining the icon’s meaning.

Best practices:

- Keep text brief
- Focus on one key idea
- Avoid long paragraphs

---

## Layout Behavior

### Responsive Grid

The icon grid adjusts based on screen size:

- **Desktop:** Up to four icons per row
- **Tablet:** Two icons per row
- **Mobile:** Icons stack vertically

This ensures the content remains readable across devices.

---

### Center Alignment

All icon items are centered to maintain visual consistency:

- Icons appear above text
- Headings and paragraphs are centered
- Spacing between items is evenly distributed

---

## Background Styling

The block includes a decorative background pattern that spans the entire section.

This pattern:

- Covers the full width of the page
- Provides visual separation from surrounding sections
- Maintains consistent branding across pages

Editors do not need to configure the background.

---

## Accessibility

The Icon Section block supports accessibility best practices:

- Icon images include alt text
- Content uses semantic headings
- Text remains readable across devices
- Buttons in the context area are keyboard accessible

---

## Tips for Best Results

- Use **consistent icon styles** across all items
- Keep headings short and descriptive
- Limit paragraph text to one or two sentences
- Avoid adding too many items to keep the section readable
- Use the context section to clearly introduce the purpose of the icons
